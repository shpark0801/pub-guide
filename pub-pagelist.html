01. 항목 작성 기준

산출물 리스트 항목별 작성 방법
항목 작성 기준 예시
1~4Depth IA 순서에 따라 상위에서 하위 메뉴 순으로 작성한다. 컴포넌트 > UI > Button
화면 ID 별도의 화면 ID를 사용하지 않는 경우 하이픈으로 표시한다. -
페이지명 브라우저 제목이 아닌 화면의 대표 명칭을 작성한다. Button / Tab
페이지 경로 현재 파일에서 접근 가능한 상대 경로를 링크로 연결한다. ui.html
상태 대기, 진행, 검수, 완료 중 현재 상태 하나를 표시한다. 진행
최종 수정일 페이지 내용이 마지막으로 변경된 날짜를 YYYY-MM-DD로 작성한다. 2026-07-15
비고 신규·수정·참조 이력을 날짜와 핵심 내용으로 구분해 최신순으로 작성한다. 수정 2026-07-15 버튼 정렬 보완

02. 이력 배지

  • 신규 페이지나 기능을 새로 추가한 경우
  • 수정 마크업, 스타일, 기능을 변경한 경우
  • 참조 연관 페이지, 개발 조건, 확인 사항을 안내하는 경우
  • 동일 날짜의 작은 수정은 한 항목으로 묶고, 영향 범위를 알아볼 수 있게 작성한다.
  • 단순히 “수정”이라고 작성하지 않고 변경한 컴포넌트나 기능을 명시한다.
  • 최종 수정일은 가장 최근 이력의 날짜와 일치해야 한다.

03. 상태별 클래스

  • 대기 .pub-status-wait
  • 진행 .pub-status-progress
  • 검수 .pub-status-review
  • 완료 .pub-status-complete
  • 각 상태 클래스는 공통 .pub-status 와 함께 사용한다.
