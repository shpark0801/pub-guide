05. button - 텍스트

  • 텍스트 버튼도 키보드 초점 표시와 충분한 클릭 영역을 유지한다.

01. tab - 라인

상세정보 내용
리뷰 내용 
<!-- line tab 예제 -->
<div class="ui-tab ui-tab-line js-tab">
    <div class="ui-tab-list" role="tablist" aria-label="상품 정보">
        <button type="button" id="tab-line-01" class="ui-tab-button" role="tab" aria-controls="panel-line-01" aria-selected="true">상세정보</button>
        <button type="button" id="tab-line-02" class="ui-tab-button" role="tab" aria-controls="panel-line-02" aria-selected="false" tabindex="-1">리뷰</button>
    </div>
    <div id="panel-line-01" class="ui-tab-panel" role="tabpanel" aria-labelledby="tab-line-01">상세정보 내용</div>
    <div id="panel-line-02" class="ui-tab-panel" role="tabpanel" aria-labelledby="tab-line-02" hidden>리뷰 내용</div>
</div>
// tab 동작
document.querySelectorAll('.js-tab').forEach(function (tab) {
    const buttons = tab.querySelectorAll('[role="tab"]');
    const panels = tab.querySelectorAll('[role="tabpanel"]');

    buttons.forEach(function (button, index) {
        button.addEventListener('click', function () {
            buttons.forEach(function (item) { item.setAttribute('aria-selected', 'false'); });
            panels.forEach(function (panel) { panel.hidden = true; });
            button.setAttribute('aria-selected', 'true');
            panels[index].hidden = false;
        });
    });
});

accordion

01. 기본형

  • 연관된 내용을 제목별로 구분하고 사용자가 필요한 내용을 선택하여 확인할 때 사용합니다.

  • 제목 버튼의 aria-controls 값과 콘텐츠 패널의 id 값을 동일하게 작성합니다.

  • 버튼의 aria-expanded와 패널의 hidden 상태를 실제 화면 상태와 동일하게 변경합니다. 

<!-- 아코디언 리스트 -->
<ul class="ui-accordion js-accordion">
    <li class="ui-accordion-item">
        <h4>
            <button
                type="button"
                class="ui-accordion-button"
                aria-expanded="false"
                aria-controls="accordion-panel-01"
            >
                아코디언은 언제 사용하나요?
            </button>
        </h4>
        <div id="accordion-panel-01" class="ui-accordion-panel" hidden>
            <p>연관된 내용을 제목별로 구분하여 확인할 때 사용합니다.</p>
        </div>
    </li>
    <li class="ui-accordion-item">
        <h4>
            <button
                type="button"
                class="ui-accordion-button"
                aria-expanded="false"
                aria-controls="accordion-panel-02"
            >
                제목과 패널은 어떻게 연결하나요?
            </button>
        </h4>
        <div id="accordion-panel-02" class="ui-accordion-panel" hidden>
            <p>aria-controls 값과 패널의 id 값을 동일하게 작성합니다.</p>
        </div>
    </li>
    <li class="ui-accordion-item">
        <h4>
            <button
                type="button"
                class="ui-accordion-button"
                aria-expanded="false"
                aria-controls="accordion-panel-03"
            >
                펼침 상태는 어떻게 표시하나요?
            </button>
        </h4>
        <div id="accordion-panel-03" class="ui-accordion-panel" hidden>
            <p>aria-expanded와 hidden 상태를 실제 화면 상태와 동일하게 변경합니다.</p>
        </div>
    </li>
</ul>
  • 여러 항목으로 구성된 리스트형 아코디언은 ul과 li로 작성한다.
  • 리스트형은 새 항목을 펼치면 기존에 펼쳐진 항목을 접는다.
  • 하나의 항목만 사용하는 단일 아코디언은 div로 작성한다.
  • 각 항목의 제목 버튼과 콘텐츠 패널을 고유한 aria-controls와 id로 연결한다.
  • 버튼의 aria-expanded와 패널의 hidden 상태를 함께 변경한다.
  • 여러 항목에서 콘텐츠 높이가 달라도 모션 종료 후 내용이 잘리지 않는지 확인한다.
  • 콘텐츠 높이 변화가 큰 경우 prefers-reduced-motion 설정에서 펼침·접힘 모션을 즉시 완료한다.

02. 단일형

하나의 제목과 콘텐츠만 펼치고 접어야 할 때 사용합니다. 

<!-- 단일 아코디언 -->
<div class="ui-accordion js-accordion">
    <div class="ui-accordion-item">
        <h4>
            <button
                type="button"
                class="ui-accordion-button"
                aria-expanded="false"
                aria-controls="accordion-single-panel-01"
            >
                단일 아코디언은 언제 사용하나요?
            </button>
        </h4>
        <div id="accordion-single-panel-01" class="ui-accordion-panel" hidden>
            <p>하나의 제목과 콘텐츠만 펼치고 접어야 할 때 사용합니다.</p>
        </div>
    </div>
</div>
  • 하나의 항목만 사용하는 단일형은 div로 작성한다.
  • 단일형은 리스트형의 단일 펼침 제어 대상에 포함하지 않는다.
  • 모션 축소 설정에서도 aria-expanded와 hidden 상태는 실제 노출 결과와 동일하게 변경한다.

pagination

01. 페이지 버튼 유형

이전 1 2 다음

‹ 1 2 ›

1 2 
<!-- pagination 기호형 -->
<nav class="ui-pagination" aria-label="페이지 이동">
    <a href="?page=1" class="ui-pagination-symbol" aria-label="이전 페이지"><span aria-hidden="true">‹</span></a>
    <a href="?page=1" aria-current="page">1</a>
    <a href="?page=2" class="ui-pagination-symbol" aria-label="다음 페이지"><span aria-hidden="true">›</span></a>
</nav>
  • 현재 페이지는 aria-current="page"로 표시한다.
  • 기호나 이미지만 제공하는 이동 버튼에는 동작을 설명하는 aria-label을 반드시 작성한다.

tooltip

01. 기본형

추가 안내 내용입니다. 
<!-- tooltip 기본형 -->
<span class="ui-tooltip">
    <button type="button" aria-describedby="tooltip-01">기본 툴팁</button>
    <span id="tooltip-01" class="ui-tooltip-text" role="tooltip">추가 안내 내용입니다.</span>
</span>

02. 말풍선형

말풍선 형태의 안내입니다. 
<!-- tooltip 말풍선형 -->
<span class="ui-tooltip ui-tooltip-bubble">
    <button type="button" aria-describedby="tooltip-02">말풍선 툴팁</button>
    <span id="tooltip-02" class="ui-tooltip-text" role="tooltip">말풍선 형태의 안내입니다.</span>
</span>
  • 기본형은 간단한 보조 설명, 말풍선형은 트리거와 설명의 연결을 시각적으로 강조할 때 사용한다.
  • 예제 버튼의 색상은 유형 구분을 위한 것이며 실제 화면에서는 주변 버튼 체계를 따른다.
  • 마우스 hover와 키보드 focus에서 동일한 내용을 확인할 수 있어야 하며 Esc로 닫을 수 있어야 한다.
  • 중요한 정보나 오류 문구는 tooltip에만 제공하지 않는다.

03. 노출 위치

시작 경계에 맞춘 안내 가운데에 맞춘 안내 끝 경계에 맞춘 안내 하단에 배치한 안내 
<span class="ui-tooltip ui-tooltip-center">
    <button type="button" aria-describedby="tooltip-pos">가운데</button>
    <span id="tooltip-pos" class="ui-tooltip-text" role="tooltip">안내 내용</span>
</span>
  • 기본은 텍스트 시작 경계이며, 중앙·끝·하단 배치가 필요할 때 위치 클래스만 추가한다.
  • 화면 가장자리에서는 안내가 잘리지 않는지 실제 문구 길이로 확인한다.
  • 상위 요소에 overflow: hidden 이 적용되어 안내 영역이 잘리지 않는지 확인한다.
