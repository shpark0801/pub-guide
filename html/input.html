input

01. Textinput - 라인형 

<div class="form-field">
    <label for="input-line-default" class="form-label">기본 상태</label>
    <input
        type="text"
        id="input-line-default"
        class="form-input form-input-line"
        placeholder="기본 상태"
    />
</div>
Input은 .form-field를 기본 구조로 사용하며, 스타일은 아래 클래스로 구분한다.
라인형 : .form-input-line
박스형 : .form-input-box

02. Textinput - 라인형 상태별

Textinput - 라인형 + 에러

입력값을 확인해주세요. 

<div class="form-field is-error">
    <label for="input-line-error" class="form-label">에러 상태</label>
    <input
        type="text"
        id="input-line-error"
        class="form-input form-input-line"
        placeholder="에러 상태"
        aria-invalid="true"
        aria-describedby="input-line-error-message"
    />
    <p id="input-line-error-message" class="form-msg form-msg-error">
        입력값을 확인해주세요.
    </p>
</div>

Textinput - 라인형 + 경고

입력 형식을 다시 확인해주세요. 


<div class="form-field is-warn">
    <label for="input-line-warning" class="form-label">경고 상태</label>
    <input
        type="text"
        id="input-line-warning"
        class="form-input form-input-line"
        placeholder="경고 상태"
        aria-describedby="input-line-warning-message"
    />
    <p id="input-line-warning-message" class="form-msg form-msg-warn">
        입력 형식을 다시 확인해주세요.
    </p>
</div>
input의 상태 유형은 부모 요소의 상태 클래스로 구분된다.
에러 : .is-error
경고 : .is-warn

Textinput - 라인형 + 비활성 (예시)


<div class="form-field">
    <label for="input-line-disabled" class="form-label">비활성 상태</label>
    <input
        type="text"
        id="input-line-disabled"
        class="form-input form-input-line"
        value="입력할 수 없습니다"
        disabled
    />
</div>
input에 disabled 속성을 직접 적용하며, .form-input:disabled 선택자로 비활성 스타일이 적용된다. 일반 input은 별도의 상태 클래스가 필요하지 않다.

03. Textinput - 플로팅 라벨 (예시)

이름을 입력해 주세요. 

<div class="form-field form-float is-error">
    <label for="input-float-required" class="form-label form-float-label">
        <span class="form-float-text">
            이름
            <span class="form-required" aria-hidden="true">*</span>
            <span class="blind">필수</span>
        </span>
    </label>
    <input
        type="text"
        id="input-float-required"
        class="form-input form-input-box"
        placeholder="이름을 입력해 주세요."
        aria-invalid="true"
        aria-describedby="input-float-required-error"
        required
    />
    <p id="input-float-required-error" class="form-msg form-msg-error">
        이름을 입력해 주세요.
    </p>
</div>
label은 12px로 표시하며 PC와 모바일에서 모두 입력 테두리의 왼쪽 상단에 겹쳐 배치한다. 흰색 배경은 label 전체가 아닌 텍스트 영역에만 적용한다.
필수 입력은 required 속성을 사용하고, 별표는 장식으로 숨긴 뒤 화면낭독기용 필수 문구를 함께 제공한다.

플로팅 라벨 + 비활성 (예시)

<div class="form-field form-float is-disabled">
    <label for="input-float-disabled" class="form-label form-float-label">
        <span class="form-float-text">이름</span>
    </label>
    <input
        type="text"
        id="input-float-disabled"
        class="form-input form-input-box"
        placeholder="이름을 입력해 주세요."
        disabled
    />
</div>
disabled input은 label 텍스트 배경도 비활성 배경색으로 전환된다.
:has(.form-input:disabled) 선택자로 disabled input을 포함한 플로팅 영역을 감지한다.
:has()를 지원하지 않는 환경에서는 .form-float에 .is-disabled 클래스를 함께 사용한다.

04. Textinput - 반응형 플로팅 라벨 (예시) 

<div class="form-row form-row-float">
    <label for="input-row-float-required" class="form-row-label">
        <span class="form-float-text">
            이메일
            <span class="form-required" aria-hidden="true">*</span>
            <span class="blind">필수</span>
        </span>
    </label>
    <div class="form-row-control">
        <input
            type="email"
            id="input-row-float-required"
            class="form-input form-input-box"
            placeholder="이메일을 입력해 주세요."
            required
        />
    </div>
</div>
PC에서는 label을 입력 영역 왼쪽에 배치하고, 768px 이하에서는 12px label이 입력 테두리의 왼쪽 상단에 겹치는 형태로 전환된다.
비활성 적용 시 부모 요소에 .is-disabled를 추가하여 .form-row.form-row-float.is-disabled로 사용하고,
input에는 disabled 속성을 적용한다. 모바일 겹침 상태에서도 label 텍스트 배경이 비활성 배경색으로 전환된다.

05. 브라우저 자동완성 스타일 (참고)

Chrome, Edge, Safari 등 일부 브라우저는 자동완성된 input에 브라우저 기본 배경색과 글자색을 적용할 수 있다. 디자인 유지가 필요한 경우에만 다음 스타일을 참고하여 선택적으로 사용한다. 
/* Chrome·Edge 등 자동완성 배경 대응 */
.form-input:-webkit-autofill,
.form-input:-webkit-autofill:hover,
.form-input:-webkit-autofill:focus,
.form-input:-webkit-autofill:active {
    -webkit-box-shadow: 0 0 0 1000px #fff inset;
    -webkit-text-fill-color: #222;
    caret-color: #222;
    transition: background-color 9999s ease-out;
}
  • 필수 공통 스타일이 아니며 자동완성 배경이 디자인과 충돌할 때만 사용한다.
  • 선택자는 전체 input이 아닌 적용 대상 클래스 범위로 제한한다.
  • inset 색상과 글자색, caret 색상은 실제 입력 영역의 배경색과 디자인 색상에 맞게 변경한다.
  • 긴 transition은 브라우저 기본 배경의 재적용을 늦추는 우회 방식이므로 프로젝트에서 상태 전환에 영향이 없는지 확인하고 불필요하면 제외한다.
  • 자동완성 기능 자체를 제거하지 않으며 autocomplete 속성은 입력 목적에 맞게 제공한다.

textarea

01. 기본형

최대 500자까지 입력할 수 있습니다. 

<!-- textarea 기본형 -->
<label class="form-label" for="textarea-basic">문의 내용</label>
<textarea id="textarea-basic" class="form-textarea" rows="5" maxlength="500" aria-describedby="textarea-help"></textarea>
<p id="textarea-help" class="form-msg">최대 500자까지 입력할 수 있습니다.</p>
label을 연결하고 글자 수 제한이나 오류 안내는 aria-describedby로 제공한다.

file

01. 기본형

JPG, PNG, PDF 파일만 첨부할 수 있습니다. 

<!-- file 기본형 -->
<input id="file-basic" class="form-file" type="file" accept=".jpg,.jpeg,.png,.pdf" />

02. 버튼 좌측 / 우측 


<!-- file 버튼 우측형 -->
<div class="form-file-group">
    <input id="file-name" class="form-file-name" type="text" readonly aria-label="선택된 파일명" />
    <input
        id="file-select"
        class="form-file-input js-file"
        type="file"
        data-file-name="#file-name"
    />
    <label class="form-file-button" for="file-select">파일 선택</label>
</div>

03. 박스형 / 아이콘형

파일을 선택해 첨부해 주세요.


<!-- file 아이콘 버튼형 -->
<input id="file-icon" class="form-file-input" type="file" />
<label class="form-file-button form-file-icon" for="file-icon" aria-label="파일 선택">
    <svg aria-hidden="true" focusable="false">...</svg>
</label>
  • 파일 input은 display:none으로 제거하지 않고 label을 통해 키보드와 보조기기에서 선택할 수 있게 연결한다.
  • 아이콘 단독 선택 버튼에는 aria-label로 동작명을 제공한다.
  • accept는 선택 보조 수단이므로 파일 형식과 용량은 서버에서도 검증한다.
